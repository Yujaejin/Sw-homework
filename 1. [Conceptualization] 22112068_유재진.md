[그림 1-1\] 최근 5년간 ADHD 진료현황

 국민건강보험공단으로부터 제출받은 '최근 5년간 ADHD 진료현황'에 따르면
현대 사회에서 스마트폰, SNS, 다양한 디지털 콘텐츠의 증가로 인해 학습 중
집중력이 쉽게 분산되는 문제가 발생하고 있다. 특히 장시간 공부를 하려고
할수록 오히려 집중력이 떨어지고, 비효율적인 학습이 이루어지는 경우가
많다.

 이러한 뉴스기사 자료뿐만 아니라 3년 만에 복학하여 대학 공부를 다시 하게
된 필자 또한, 이전과 달리 집중력이 쉽게 흐트러진다는 문제를 체감하게
되었다. 이를 해결하고자 관련 내용을 찾아보던 중 집중력 저하를 스스로
해결하는 데에 효과적인 방법의 하나로 일정 시간 동안 집중하고 짧은 휴식을
반복하는 포모도로 기법이 있음을 알게 되었으며 이 포모도로 기법을
적용하여 사용자의 중력을 유지하고 학습 효율을 높이기 위해 타이머
소프트웨어를 설계하고자 마음먹게 되었다.

### 1.2 Goal
 본 소프트웨어의 목표는 사용자가 일정한 시간 동안 집중하여 학습할 수
있도록 돕고, 규칙적인 휴식을 통해 집중력을 지속적으로 유지하도록
지원하는 데에 있다.

 이를 위해 첫째, 사용자가 공부 시간과 휴식 시간을 명확하게 구분하여
사용할 수 있도록 타이머 기능을 제공한다.

 둘째, 반복적인 학습 사이클을 통해 집중과 휴식의 균형을 유지하도록 한다.

 셋째, 사용자의 학습 패턴을 개선하여 장기적으로 효율적인 공부 습관을
형성하는 것을 목표로 한다.

 결과적으로 본 프로그램은 단순한 타이머를 넘어, 사용자의 집중력을
향상하고 자기 주도적 학습 능력을 강화하는 데 이바지하는 것을 목표로
한다.

## **4. Concept of operation**
<br/>
#### 1\) Sign up -- 회원가입
  --------------- -------------------------------------------------------
  Purpose   -      사용자가 서비스 이용을 위해 계정을 생성할 수 있도록
                  한다.

  Approach  -      사용자가 아이디 및 비밀번호를 입력하면 시스템 이용을
                  위한 권한을 부여하고 서버가 이를 검증 후 데이터베이스에
                  저장한다.

  Dynamics  -      사용자가 서비스 이용 전 회원가입을 통해 권한을
                  부여받아야 할 경우

  Goals     -      사용자별 데이터 관리를 위한 계정을 생성한다.
  --------------- -------------------------------------------------------
<br/>

#### 2\) Login/Logout -- 로그인/로그아웃
  --------------- -------------------------------------------------------
  Purpose   -      사용자가 자신의 계정으로 시스템에 접근하거나 로그아웃할
                  수 있도록 한다.

  Approach    -    로그인 시 입력창을 통해 ID, PW를 입력받고 이를 서버에
                  전달하여 검증한 후 로그인. 실패 시 오류 메시지를
                  출력하며, 로그아웃 시 세션을 종료한다.

  Dynamics  -      사용자가 개인의 계정으로 시스템을 이용하고 싶거나
                  종료하고 싶은 경우

  Goals    -       사용자 인증 및 개인 데이터를 보호한다.
  --------------- -------------------------------------------------------
<br/>

#### 3\) Set Focus Time -- 집중시간 설정
  --------------- -------------------------------------------------------
  Purpose   -      사용자가 원하는 집중시간을 설정할 수 있도록 한다.

  Approach -       사용자 입력 값을 서버로 전달하여 DB에 저장한다.

  Dynamics  -      사용자가 기본값(25분) 외에 다른 값을 정하고 싶은 경우

  Goals    -       사용자가 직접 원하는 시간만큼 집중시간을 설정해 DB에
                  저장하고 이후 시행되는 집중 타이머에 적용한다.
  --------------- -------------------------------------------------------
<br/>

#### 4\) Set Break Time -- 휴식 시간 설정
  --------------- -------------------------------------------------------
  Purpose  -       사용자가 집중 후 적용될 휴식 시간을 설정할 수 있도록
                  한다.

  Approach   -     사용자 입력 값을 서버로 전달하여 DB에 저장한다.

  Dynamics  -      사용자가 기본값(5분) 외에 다른 값을 정하고 싶은 경우

  Goals     -      사용자가 직접 원하는 시간만큼 휴식 시간을 설정해 DB에
                  저장하고 이후 시행되는 휴식 타이머에 적용한다.
  --------------- -------------------------------------------------------
<br/>

#### 5\) Start Timer -- 타이머 시작
  --------------- -------------------------------------------------------
  Purpose   -      설정된 시간에 따라 집중 타이머를 시작한다.

  Approach  -      서버 또는 클라이언트에서 DB에 저장된 시간만큼 타이머를
                  실행하고 세션을 생성한다.

  Dynamics  -      사용자가 타이머 시작을 요청할 경우

  Goals    -       사용자가 집중 타이머를 시작할 수 있게 한다.
  --------------- -------------------------------------------------------
<br/>

#### 6\) Pause / Resume Timer -- 타이머 일시 정지 / 재개
  --------------- -------------------------------------------------------
  Purpose   -      사용자가 진행 중인 타이머를 일시적으로 멈추고, 이후
                  다시 이어서 실행할 수 있도록 한다.

  Approach  -      DB에 현재 타이머 상태를 저장하고 현재 상태를 유지한 채
                  타이머를 멈추며 재개를 요청할 경우 저장된 상태를
                  기반으로 타이머를 다시 실행한다.

  Dynamics  -      사용자가 타이머를 잠시 멈추고 싶은 경우

  Goals   -        사용자가 집중 타이머를 멈추고 다시 시행함으로 유연하게
                  집중시간을 관리할 수 있도록 지원한다.
  --------------- -------------------------------------------------------
<br/>

#### 7\) Stop Timer -- 타이머 종료
  --------------- -------------------------------------------------------
  Purpose    -     타이머를 종료하고 세션을 종료 상태로 기록한다.

  Approach   -     종료 시점의 데이터를 계산하여 세션을 저장한다.

  Dynamics  -      사용자가 타이머 종료를 요청할 경우

  Goals      -     사용자가 타이머를 종료할 수 있게 만든다.
  --------------- -------------------------------------------------------
<br/>

#### 8\) View Personal Statistics -- 개인 통계 보기
  --------------- -------------------------------------------------------
  Purpose   -      사용자가 자신의 집중 데이터를 확인할 수 있도록 한다.

  Approach  -      서버에서 사용자별 세션 데이터를 분석하여 집중시간 통계
                  및 시행 횟수 등의 결과를 제공한다.

  Dynamics  -      사용자가 자신의 타이머 사용 통계 조회를 요청할 경우

  Goals     -      사용자의 통계 데이터를 제공하도록 한다.
  --------------- -------------------------------------------------------
<br/>

#### 9\) View Personal Statistics -- 개인 통계 보기
  --------------- -------------------------------------------------------
  Purpose    -     전체 사용자 기준의 집중 순위를 확인할 수 있도록 한다.

  Approach   -     사용자들의 데이터를 집계하여 순위를 계산 후 사용자의
                  순위를 제공한다.

  Dynamics    -    사용자가 자신의 집중시간 랭킹을 조회하고 싶은 경우

  Goals       -    사용자의 전체 사용자 기준의 통계 데이터를 제공하도록
                  한다.
  --------------- -------------------------------------------------------
<br/>

#### 10\) Save User Data -- 사용자 설정 저장
  --------------- -------------------------------------------------------
  Purpose   -      사용자 개인 설정 및 세션 데이터를 DB에 저장한다.

  Approach  -      사용자가 설정한 집중시간, 휴식 시간 및 세션 데이터를
                  DB에 저장한다.

  Dynamics  -      사용자의 설정이나 세션 데이터가 변경될 경우

  Goals    -       사용자가 한 번 설정한 값들을 지속해서 사용할 수 있도록
                  하고, 사용자의 타이머 사용 통계를 DB에 저장한다.
  --------------- -------------------------------------------------------
<br/>

#### 11\) Load User Data -- 사용자 설정 불러오기
  --------------- -------------------------------------------------------
  Purpose   -      저장된 사용자 데이터를 불러와 환경을 복원한다.

  Approach -       저장되어있는 사용자 데이터를 서버가 DB에서 데이터를
                  조회하여 사용자에게 전달한다.

  Dynamics  -      사용자가 시스템에 접속하거나 데이터를 요청할 경우

  Goals     -      사용자 개인의 맞춤 환경을 유지할 수 있다.
  --------------- -------------------------------------------------------

## **5. Problem statement**
<br/>
\- Overview
'Focus Tracker' 는 사용자가 직접 설정한 집중시간과 휴식 시간을
반복적으로 알림 해주는 타이머를 제공하고, 이를 세션 데이터로 저장 및
분석하여 시각적으로 보여주는 기능을 가진다. 또한, 전체 사용자들의 랭킹을
제공함으로써 동기 부여와 경쟁 요소를 강화하고, 궁극적으로 사용자의
집중력 향상을 돕는 것을 목적으로 한다.

해당 소프트웨어를 만드는 과정에서 예상되는 기술적 어려움으로는 앱 개발
경험 및 서버, 데이터베이스와 같은 백엔드 개발 경험이 부족하여 전체
시스템을 설계하고 구현하는 데 어려움이 예상된다. 특히 프론트엔드와
백엔드를 연동하고 데이터를 안정적으로 처리하는 과정에서 학습 및 구현
부담이 존재한다. 따라서 개발 과정에 많은 시간을 할애할 것이다.

또한, 랭킹 및 통계 처리 시 서버 부하 문제로 다수 사용자의 세션 데이터를
기반으로 랭킹 및 통계를 계산할 경우 서버의 부하가 증가할 수 있다. 특히
실시간으로 순위를 갱신하거나 통계를 처리할 때 성능 저하가 발생할
가능성이 있어 이를 효율적으로 처리하는 방법을 고민해야 할 것이다.

다음으로 'Focus Tracker'를 단순한 기능 제공을 넘어, 안정적이고 효율적인
사용자 경험을 보장하는 시스템으로 설계하기 위해 시스템의 성능과 사용성을
동시에 고려하여 비기능적 요구사항을 정의하였다.

1\. 앱 초기 구동 속도는 2초 이내여야 한다.

2\. 개인 통계 및 전체 랭킹을 요청했을 때 3초 내로 제공하여야 한다.

3\. 앱 내 모든 기능은 별도의 교육 없이 3분 이내에 학습 가능해야 한다.
