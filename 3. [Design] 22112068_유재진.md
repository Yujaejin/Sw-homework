## **1. Introduction**

&ensp;현대의 대학생들은 학업, 과제, 시험 준비 등 다양한 활동을 수행해야 하지만 스마트폰, SNS, 영상 플랫폼 등 여러 외부 요인으로 인해 집중력을 유지하는 데 어려움을 겪는 경우가 많다. 특히 본 프로젝트의 개발자는 3년 만에 복학한 이후 학습에 다시 적응하는 과정에서 장시간 집중하는 것이 쉽지 않다는 점을 체감하였다. 이러한 문제는 학습 효율 저하와 시간 관리의 어려움으로 이어질 수 있다.

Focus Tracker는 이러한 문제를 해결하기 위해 포모도로 기법(Pomodoro Technique)을 기반으로 설계된 집중 시간 관리 애플리케이션이다. 사용자는 집중 시간(Focus Time)과 휴식 시간(Break Time)을 직접 설정할 수 있으며, 타이머 기능을 통해 체계적으로 학습 시간을 관리할 수 있다. 또한 집중 세션 기록을 데이터로 저장하고 분석하여 개인 통계를 제공하며, 전체 사용자 랭킹 기능을 통해 동기 부여와 경쟁 요소를 제공한다.

이를 통해 사용자는 자신의 학습 패턴을 객관적으로 확인할 수 있으며, 지속적인 집중 습관 형성과 생산성 향상을 기대할 수 있다.

&ensp;이 문서는 Focus Tracker를 개발하기 위한 세 번째 단계인 Design에 대한 내용으로, 실제 구현을 위한 시스템 구조와 클래스 설계, 객체 간 상호작용, 상태 변화 및 사용자 인터페이스를 다이어그램을 통해 시각적으로 표현한다.

<그림 3-1>

&ensp;그림 3-1는 사용자가 Focus Tracker에 로그인하는 과정을 나타낸 Sequence Diagram이다.
사용자는 로그인 화면에서 이메일과 비밀번호를 입력한 후 로그인 버튼을 선택한다. Login 객체는 입력받은 정보를 ServerManager에 전달하여 사용자 인증을 요청한다. ServerManager는 DatabaseManager를 통해 저장된 사용자 정보를 조회하고 입력값과 비교하여 인증을 수행한다. 

인증 결과가 성공인 경우 사용자에게 로그인 성공 메시지를 반환하고 메인 화면으로 이동하며, 실패한 경우 로그인 실패 메시지를 출력한다.

<그림 3-2>

&ensp;그림 3-2는 사용자가 집중 세션을 시작하고 세션 기록이 저장되는 과정을 나타낸 Sequence Diagram이다.
사용자가 타이머 시작 버튼을 선택하면 Timer 객체가 TimerSettings에 저장된 집중 시간을 불러온다. 이후 타이머가 동작하며 설정된 집중 시간이 종료되면 Notification 객체를 통해 사용자에게 알림을 전송한다. 

알림 전송 후 Timer는 SessionTracking 객체에 세션 정보를 전달하고 새로운 SessionData를 생성한다. 생성된 세션 데이터는 DatabaseManager를 통해 데이터베이스에 저장되며, 이를 통해 사용자의 집중 기록이 누적 관리된다.

&ensp;그림 3-3는 사용자가 개인 통계를 조회하는 과정을 나타낸 Sequence Diagram이다.
사용자가 개인 통계 화면을 요청하면 PersonalStatistics 객체가 Statistics 객체에 통계 계산을 요청한다. Statistics는 SessionTracking으로부터 사용자의 세션 기록을 조회하며, 필요한 경우 DatabaseManager를 통해 저장된 세션 데이터를 불러온다. 

이후 총 집중 시간, 세션 수, 평균 집중 시간 등의 통계를 계산하고 결과를 PersonalStatistics 객체에 전달한다. 최종적으로 사용자는 그래프 및 수치 형태로 자신의 집중 기록을 확인할 수 있다.

사용자가 전체 랭킹 화면을 요청하면 GlobalRanking 객체가 ServerManager에 랭킹 데이터를 요청한다. ServerManager는 DatabaseManager를 통해 전체 사용자 데이터를 조회하고 누적 집중 시간을 기준으로 순위를 계산한다. 

계산된 랭킹 결과는 GlobalRanking 객체로 전달되며, 최종적으로 사용자는 자신의 순위와 다른 사용자들의 집중 기록 순위를 확인할 수 있다. 이를 통해 사용자에게 학습 동기 부여와 경쟁 요소를 제공한다.

<그림 4-1>  
 
<br/>

&ensp;<그림 4-1>은 사용자가 Focus Tracker 시스템에 접근하여 인증을 수행하는 과정에서 발생하는 상태 변화를 나타낸다.

시스템은 초기 상태인 HomeScreen에서 시작하며, 사용자는 로그인(Login) 또는 회원가입(Sign Up)을 선택할 수 있다. 회원가입을 선택한 경우 사용자 정보 입력 및 검증 과정을 거쳐 AccountCreated 상태로 전이되며, 이후 로그인 화면으로 이동한다.

로그인 화면에서 사용자가 이메일과 비밀번호를 입력하면 Authentication 상태로 전이되어 서버와 데이터베이스를 통해 사용자 인증을 수행한다. 인증에 성공하면 LoggedIn 상태로 전이되어 시스템의 모든 기능을 사용할 수 있으며, 인증에 실패하면 LoginFailed 상태로 이동하여 오류 메시지를 출력한 후 다시 로그인 화면으로 복귀한다.

로그인된 사용자는 언제든지 Logout 기능을 수행할 수 있으며, 로그아웃 시 사용자 세션이 종료되고 시스템은 다시 HomeScreen 상태로 전환된다.

<p align="center">  
<그림 4-2>  

<br/>

&ensp;<그림 4-2>은 Focus Tracker의 핵심 기능인 포모도로 타이머의 동작 과정에서 발생하는 상태 변화를 나타낸다.

타이머는 초기 상태인 Ready 상태에서 시작된다. 사용자가 Start 버튼을 누르면 FocusRunning 상태로 전이되어 집중 시간이 진행된다. 집중 시간 진행 중 사용자는 Pause 기능을 이용하여 FocusPaused 상태로 전환할 수 있으며, Resume 기능을 통해 다시 집중 시간을 이어서 진행할 수 있다.

집중 시간이 모두 경과하면 FocusCompleted 상태로 전이되며, 집중 완료 알림이 발생한 후 자동으로 BreakRunning 상태가 시작된다. 휴식 시간 역시 Pause와 Resume 기능을 사용할 수 있으며, 휴식 시간이 종료되면 BreakCompleted 상태로 전이된다.

집중 및 휴식 세션이 완료되면 시스템은 SaveSession 상태에서 세션 정보를 저장하고, 이후 UpdateStatistics 상태에서 사용자의 총 집중 시간, 세션 수, 평균 집중 시간 등의 통계 정보를 갱신한다. 통계 갱신이 완료되면 다시 Ready 상태로 복귀하여 새로운 포모도로 세션을 시작할 수 있다.

또한 사용자가 집중 시간 또는 휴식 시간 진행 중 Stop 버튼을 누를 경우 현재 세션은 강제로 종료되며, 진행된 기록을 저장한 후 통계 정보를 갱신하고 Ready 상태로 복귀한다.

## **5. Implementation requirements**  

## 5.1. H/W Platform Requirements

- CPU : Intel Core i3 이상
- RAM : 4GB 이상
- Storage : 1GB 이상 여유 공간
- Network : 인터넷 연결 필요 (로그인, 통계 조회, 랭킹 조회 기능 사용 시)


## 5.2. S/W Platform Requirements

- OS : Android 10 이상
- Development Environment : Android Studio
- Implementation Language : Java

## **6. Glossary**  

| Term | Description |
|------|-------------|
| Focus Tracker | 사용자의 집중 시간 관리와 학습 습관 형성을 지원하는 포모도로 타이머 애플리케이션 |
| User | Focus Tracker를 이용하는 사용자 |
| Focus Time | 사용자가 작업이나 학습에 집중하는 시간 |
| Break Time | 집중 세션 종료 후 제공되는 휴식 시간 |
| Session | 하나의 집중 시간과 휴식 시간으로 구성된 작업 단위 |
| Timer | Focus Time과 Break Time을 측정하고 관리하는 기능 |
| Timer Settings | 사용자가 집중 시간, 휴식 시간 및 알림 설정을 변경하는 기능 |
| Personal Statistics | 사용자의 집중 기록과 학습 패턴을 분석하여 보여주는 기능 |
| Global Ranking | 전체 사용자들의 집중 기록을 기반으로 순위를 제공하는 기능 |
| Notification | 집중 시간 또는 휴식 시간 종료 시 사용자에게 알림을 제공하는 기능 |
| Login | 등록된 사용자가 계정 인증을 통해 시스템에 접속하는 기능 |
| Sign Up | 새로운 사용자가 계정을 생성하는 기능 |
| Logout | 현재 로그인된 사용자의 세션을 종료하는 기능 |
| Database | 사용자 정보, 세션 기록, 통계 데이터를 저장하는 저장소 |
| Ranking | 누적 집중 시간을 기준으로 계산된 사용자 순위 정보 |
